# AI 프롬프트 작성 가이드

> NMWC 팀이 AI 코딩 에이전트(Cursor·Claude Code 등)와 일할 때, 프롬프트 품질로
> 결과의 품질과 속도를 끌어올리기 위한 가이드.

## 왜 프롬프트 작성법이 중요한가?

> AI Agent는 `"내가 어떻게 말하느냐"`에 따라 결과의 품질과 속도가 크게 달라진다.
> 같은 작업도 프롬프트를 어떻게 쓰느냐에 따라 **대화 횟수가 5회에서 끝날 수도 있고, 50회를 넘길 수도 있다.**
> 이 문서는 실제 Cursor 대화 로그를 분석하여, **잘 쓴 프롬프트와 개선할 수 있는 프롬프트의 차이**를 정리한 가이드다.

## 프롬프트 작성 가이드 (구성)

- 핵심 원칙 3가지
- 상황별 프롬프트 작성법
- 토큰 효율 이해하기
- 대화 길이 관리법
- 흔한 안티 패턴과 개선법
- 프롬프트 작성 체크리스트

## 요약

| 원칙 | 핵심 |
| --- | --- |
| 목표 먼저 | AI에게 "결국 뭘 해야 하는지"를 첫 문장에 알려줘라 |
| 아는 것은 알려줘라 | 프로젝트 구조, 컨벤션, 원인 추정을 프롬프트에 포함해라 |
| 한번에 구체적으로 | 여러 턴에 걸쳐 조금씩 요청하기보다, 스펙을 정리해서 한번에 전달해라 |
| 모국어 + 영어 기술 용어 | 모국어로 쓰되, 클래스명·메서드명·패턴명은 영어 그대로 유지 |
| 대화 길이 관리 | 한국어 7~8턴, 베트남어 8~9턴에서 새 대화 고려. 수정 2회 반복 시 재정리 |
| 설계 먼저, 구현 나중 | 방향이 정해지지 않은 채 "적용해줘"를 하면 수정의 연쇄가 시작된다 |
