# 스펙 주도 개발 - Claude Code (범용)

> Claude Code에서 스펙 주도 개발을 진행하는 구체적인 방법을 안내합니다.
공통 개념은 **스펙 주도 개발 - 공통 (범용)** 문서를 먼저 참고하세요.
> 

---

## 1. Claude Code와 스펙 주도 개발

Claude Code는 터미널에서 동작하는 AI 코딩 에이전트입니다. 파일 시스템에 직접 접근하여 파일을 읽고, 수정하고, 생성할 수 있습니다.

### Claude Code가 스펙 주도 개발에 유리한 점

| 특성 | 이점 |
| --- | --- |
| 파일 시스템 직접 접근 | spec.md를 별도로 복사할 필요 없이 경로만 지정하면 됨 |
| 파일 수정 가능 | PROGRESS.md를 AI가 직접 갱신할 수 있음 |
| CLAUDE.md 자동 로드 | 프로젝트 진입 시 CLAUDE.md를 자동으로 읽어 맥락 파악 |
| 스크립트 실행 | 빌드, 테스트 등을 터미널에서 바로 실행 가능 |
| 스킬 시스템 연동 | `.claude/skills/`에 스킬을 등록하면 스펙과 스킬을 함께 활용 가능 |

## 2. 환경 설정

### 2-1. 폴더 구조

```
프로젝트 루트/
├── CLAUDE.md                        # Claude Code 진입점 (자동 로드)
├── .claude/                         # Claude Code 설정
│   └── skills/                      # 스킬 폴더 (선택)
├── .specs/                          # 스펙 문서 루트
│   ├── INDEX.md
│   ├── spec_user_auth/
│   │   ├── spec.md
│   │   └── PROGRESS.md
│   └── spec_dashboard/
│       ├── spec.md
│       └── PROGRESS.md
└── src/                             # 소스 코드
    └── ...
```

### 2-2. CLAUDE.md와 스킬의 관계

| 파일 | 역할 | 로드 시점 |
| --- | --- | --- |
| `CLAUDE.md` | 프로젝트 전체 맥락 + 스펙 문서 위치 안내 | Claude Code 시작 시 자동 |
| `.claude/skills/*/SKILL.md` | 특정 작업 수행 시 참고하는 규칙 | 관련 작업 요청 시 자동 또는 수동 |
| `.specs/*/spec.md` | 개별 기능의 설계 문서 | 사용자가 지정하거나 CLAUDE.md에서 안내 |

## 3. CLAUDE.md 작성법

### 3-1. 작성 원칙

- **가볍게 유지한다.** 전체 맥락만 제공하고, 상세 내용은 스펙 문서로 안내한다.
- **개발 이력을 작성하지 않는다.** 진행 상황은 각 스펙의 PROGRESS.md에서 관리한다.
- **스펙 문서 위치를 반드시 포함한다.** Claude가 스펙 폴더의 존재를 인지할 수 있도록 한다.

### 3-2. 권장 템플릿

```markdown
# {프로젝트명}

> {프로젝트 한줄요약}

## 스펙 문서
이 프로젝트는 스펙 주도 개발(Spec-Driven Development)을 따릅니다.
- 전체 스펙 목록: `.specs/INDEX.md`
- 개별 스펙: `.specs/spec_{기능명}/spec.md`
- 진행 상황: `.specs/spec_{기능명}/PROGRESS.md`

새 작업을 시작하기 전에 반드시 해당 스펙의 spec.md를 읽고, 작업 후에는 PROGRESS.md를 갱신합니다.

## 빠른 참조
- 기술 스택: {주요 기술 나열}
- 패키지 매니저: {npm / pnpm / yarn / pip 등}
- 실행: {실행 명령어}
- 빌드: {빌드 명령어}
- 테스트: {테스트 명령어}

## 핵심 구현 포인트
- {프로젝트 전반에 걸친 중요한 규칙이나 주의사항}

## 작업 규칙
- 코드 구현이 완료되면 해당 스펙의 PROGRESS.md를 갱신합니다.
- 설계 변경이 발생하면 spec.md를 먼저 갱신하고 코드에 반영합니다.
```

```
         관련 코드에도 반영해줘."
```

## 8. 팀 공유

`.specs/` 폴더와 `CLAUDE.md`는 Git 저장소에 포함시킵니다. 팀원이 Clone하면 동일한 스펙 환경이 설정됩니다.

---

## 9. 주의사항

- **CLAUDE.md 크기 관리:** 프로젝트 전체 맥락만 간결하게 유지하고, 상세 내용은 스펙 문서로 분리합니다.
- **파일 경로 정확히 지정:** `.specs/spec_user_auth/spec.md`처럼 정확한 경로를 사용합니다.
- **PROGRESS.md 갱신 확인:** AI가 자동 갱신한 경우 커밋 전 내용을 확인합니다.
- **민감 정보:** CLAUDE.md와 스펙 문서에 API 키, 서버 주소, 토큰 등을 포함하지 않습니다.
