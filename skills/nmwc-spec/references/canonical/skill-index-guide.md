## 1. 스킬 인덱스란?

스킬 인덱스는 프로젝트에 등록된 **모든 NMWC 스킬의 목록과 상태를 한 곳에서 관리하는 문서**입니다. 스펙 주도 개발의 INDEX.md와 동일한 역할을 합니다.

스킬이 1~2개일 때는 필요성을 느끼기 어렵지만, 코딩 규약, 폴더 규약, PR 리뷰, 테스트 작성 등 스킬이 늘어나면 "지금 어떤 스킬이 있고, 각각 뭘 하는지" 파악하기 어려워집니다. 스킬 인덱스는 이 문제를 해결합니다.

---

## 2. 왜 필요한가?

| 상황 | 인덱스 없이 | 인덱스 있으면 |
| --- | --- | --- |
| 신규 입사자가 프로젝트에 합류 | 스킬 폴더를 하나씩 열어봐야 함 | 인덱스 한 페이지로 전체 파악 |
| 새 스킬을 만들고 싶을 때 | 이미 비슷한 스킬이 있는지 모름 | 목록에서 중복 여부 즉시 확인 |
| 스킬이 잘 동작하지 않을 때 | 어떤 스킬이 적용 중인지 추적 어려움 | 상태와 버전을 한눈에 확인 |
| 다른 프로젝트에 스킬을 복사할 때 | 필요한 스킬을 하나씩 찾아야 함 | 인덱스에서 필요한 것만 선택 |

## 3. 인덱스 문서 만들기

### 3-1. 파일 위치

스킬 인덱스는 AI 도구의 스킬 폴더 루트에 위치합니다.

| AI 도구 | 인덱스 파일 경로 |
| --- | --- |
| Claude Code | `프로젝트 루트/.claude/skills/INDEX.md` |
| Codex CLI | `프로젝트 루트/.codex/skills/INDEX.md` |
| Cursor | `프로젝트 루트/.cursor/rules/INDEX.md` |
| Claude Projects | Project Knowledge에 파일로 업로드 |
| OpenAI (GPTs) | Knowledge에 파일로 업로드 |

> Claude Code와 Codex CLI를 함께 사용하는 경우 심볼릭 링크로 스킬 폴더를 공유하면 인덱스도 자동으로 공유됩니다.
> 

### 3-2. 인덱스 템플릿

아래 내용을 복사하여 INDEX.md 파일을 생성하세요. 처음에는 스킬 목록이 비어있는 상태입니다.

```markdown
# NMWC 스킬 인덱스

> 이 프로젝트에 등록된 NMWC 스킬 목록입니다.

## 스킬 목록

| 스킬명 | 설명 | 상태 | 파일명 | 버전 | 최종 갱신 |
|--------|------|------|--------|------|----------|
| | | | | | |

## 상태 범례

| 상태 | 설명 |
|------|------|
| ✅ 사용중 | 현재 프로젝트에서 활성화된 스킬 |
| 🟡 검토중 | 등록 예정이거나 테스트 중인 스킬 |
| ❌ 비활성 | 일시적으로 비활성화된 스킬 |

## 갱신 규칙
- 새 스킬을 추가하면 이 파일에 즉시 등록합니다.
- 스킬이 갱신되면 버전과 최종 갱신 날짜를 함께 수정합니다.
- 비활성화된 스킬은 삭제하지 않고 상태만 변경합니다.
```

### 3-3. 스킬이 등록된 인덱스 예시

웹앱 코딩 규약과 폴더 규약 스킬이 등록된 상태의 예시입니다.

```markdown
# NMWC 스킬 인덱스

> 이 프로젝트에 등록된 NMWC 스킬 목록입니다.

## 스킬 목록

| 스킬명 | 설명 | 상태 | 파일명 | 버전 | 최종 갱신 |
|--------|------|------|--------|------|----------|
| nmwc-webapp-coding-convention | 웹앱 코딩 규약 (React+TS 네이밍, 컴포넌트, 타입) | ✅ 사용중 | SKILL.md | v1.0 | 2026-05-28 |
| nmwc-webapp-folder-convention | 웹앱 폴더 구조 및 파일 배치 규약 | ✅ 사용중 | SKILL.md | v1.0 | 2026-05-28 |
```

### 3-4. 폴더 구조 예시

Claude Code 기준으로 스킬 인덱스가 포함된 전체 구조입니다.

```
프로젝트 루트/
└── .claude/
    └── skills/
        ├── INDEX.md                              ← 스킬 인덱스
        ├── nmwc-webapp-coding-convention/
        │   └── SKILL.md
        └── nmwc-webapp-folder-convention/
            └── SKILL.md
```

Cursor 기준:

```
프로젝트 루트/
└── .cursor/
    └── rules/
        ├── INDEX.md                              ← 스킬 인덱스
        ├── webapp-coding-convention.mdc
        └── webapp-folder-convention.mdc
```

## 4. 인덱스 관리 규칙

### 4-1. 새 스킬을 추가할 때

1. 스킬 파일(SKILL.md 또는 .mdc)을 생성합니다.
2. INDEX.md의 스킬 목록 표에 새 행을 추가합니다.
3. 스킬 파일과 INDEX.md를 함께 Git 커밋합니다.

### 4-2. 스킬을 갱신할 때

1. 스킬 파일의 내용을 수정합니다.
2. INDEX.md의 해당 스킬 행에서 버전과 최종 갱신 날짜를 수정합니다.
3. 스킬 파일과 INDEX.md를 함께 Git 커밋합니다.

### 4-3. 스킬을 비활성화할 때

1. 스킬 파일을 삭제하지 않습니다.
2. INDEX.md의 해당 스킬 상태를 `❌ 비활성`으로 변경합니다.
3. 비활성 사유를 설명 칸에 간단히 표기합니다.

### 4-4. Git에 포함

INDEX.md는 스킬 파일과 함께 Git 저장소에 포함시킵니다. 팀원이 Clone하면 어떤 스킬이 등록되어 있는지 즉시 파악할 수 있습니다.
