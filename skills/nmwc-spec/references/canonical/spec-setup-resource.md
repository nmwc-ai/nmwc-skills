## 1. 이 문서는 무엇인가?

스펙 주도 개발을 시작하려면 여러 폴더와 파일을 세팅해야 합니다. 이 문서는 처음부터 만들 필요 없이, **템플릿을 복사하여 바로 사용할 수 있도록** 모든 리소스를 한 곳에 모아둔 것입니다.

### 최종 폴더 구조 미리보기

```
프로젝트 루트/
├── CLAUDE.md                            # Claude Code 사용 시
├── AGENTS.md                            # Codex CLI 사용 시
├── .claude/skills/                      # Claude Code 스킬 (선택)
├── .codex/skills/                       # Codex CLI 스킬 (선택)
├── .cursor/rules/                       # Cursor 사용 시
│   └── spec-driven-dev.mdc
├── .specs/                              # 공통 — 모든 도구에서 동일
│   └── INDEX.md
└── src/                                 # 소스 코드 (프로젝트마다 다름)
    └── ...
```

## 2. 공통 리소스

AI 도구에 관계없이 동일하게 사용하는 스펙 문서 템플릿입니다.

### 2-1. INDEX.md

`.specs/INDEX.md`로 저장합니다.

```markdown
# 스펙 인덱스

> 전체 스펙 목록 및 상태를 관리합니다.

## 스펙 목록

| 스펙 | 상태 | 스프린트 | 담당자 | 의존성 | 최종 갱신 |
|------|------|---------|--------|--------|----------|
| | | | | | |

## 상태 범례

| 아이콘 | 상태 | 설명 |
|--------|------|------|
| 🟢 | 진행중 | 현재 개발이 진행 중인 스펙 |
| 🟡 | 보류 | 일시적으로 중단된 스펙 (사유를 PROGRESS.md에 기록) |
| ✅ | 완료 | 개발 및 QA가 완료된 스펙 |
| ❌ | 폐기 | 기획 변경 등으로 폐기된 스펙 (삭제하지 않고 상태만 변경) |

## 갱신 규칙
- 새 스펙을 생성하면 이 파일에 즉시 추가합니다.
- 스펙 상태가 변경되면 이 파일을 함께 갱신합니다.
- 완료 및 폐기된 스펙은 목록 하단으로 이동시킵니다.
```

### 2-2. spec.md 템플릿

`.specs/spec_{기능명}/spec.md`로 저장합니다.

```markdown
# {기능명}

## 개요
{이 기능이 무엇이고 왜 필요한지 1~3문장으로 설명}

## 기술 스택
- {주요 기술}

## 실행 환경
- {런타임, 필요 도구 등}

## 실행 스크립트
- 실행: {실행 명령어}
- 빌드: {빌드 명령어}
- 테스트: {테스트 명령어}

## 데이터 구조

### {타입명}
| 필드 | 타입 | 설명 |
|------|------|------|
| | | |

## 파일 구성
| 파일 | 경로 | 설명 |
|------|------|------|
| | | |

## API 엔드포인트
| 메서드 | 경로 | 설명 |
|--------|------|------|
| | | |

## 변경 이력
| 날짜 | 변경 내용 |
|------|----------|
| {YYYY-MM-DD} | 최초 작성 |
```

### 2-3. PROGRESS.md 템플릿

`.specs/spec_{기능명}/PROGRESS.md`로 저장합니다.

```markdown

## 영역별 진행도

| 영역 | 상태 | 비고 |
|------|------|------|
| | ⬜ 미착수 | |

## 작업 히스토리

### {YYYY-MM-DD}
- spec.md 최초 작성

## 진행중
- [ ]

## 다음 할일
- [ ]

## 결정 기록

### {결정 제목} ({YYYY-MM-DD})
- **결정:** {무엇을 결정했는지}
- **대안:** {검토했던 대안}
- **사유:** {이 결정을 선택한 이유}
```

## 3. AI 도구별 설정 리소스

### 3-1. Claude Code — CLAUDE.md

`프로젝트 루트/CLAUDE.md`로 저장합니다.

```markdown
# {프로젝트명}

> {프로젝트 한줄요약}

## 스펙 문서
이 프로젝트는 스펙 주도 개발(Spec-Driven Development)을 따릅니다.
- 전체 스펙 목록: `.specs/INDEX.md`
- 개별 스펙: `.specs/spec_{기능명}/spec.md`
- 진행 상황: `.specs/spec_{기능명}/PROGRESS.md`

새 작업을 시작하기 전에 반드시 해당 스펙의 spec.md를 읽고, 작업 후에는 PROGRESS.md를 갱신합니다.

## 빠른 참조
- 기술 스택: {주요 기술}
- 패키지 매니저: {npm / pnpm / yarn / pip 등}
- 실행: {실행 명령어}
- 빌드: {빌드 명령어}
- 테스트: {테스트 명령어}

## 핵심 구현 포인트
- {프로젝트 전반의 핵심 규칙}

## 작업 규칙
- 코드 구현이 완료되면 해당 스펙의 PROGRESS.md를 갱신합니다.
- 설계 변경이 발생하면 spec.md를 먼저 갱신하고 코드에 반영합니다.
```

### 3-2. Codex CLI — AGENTS.md

`프로젝트 루트/AGENTS.md`로 저장합니다. 내용은 CLAUDE.md와 동일합니다.

### 3-3. Cursor — spec-driven-dev.mdc

`프로젝트 루트/.cursor/rules/spec-driven-dev.mdc`로 저장합니다.

```markdown
---
description: 스펙 주도 개발 규칙. 기능 구현, 코드 작성, 리팩토링 시 자동으로 참고됩니다.
globs:
alwaysApply: true
---

# 스펙 주도 개발 규칙

## 프로젝트 개요
{프로젝트 한줄요약}. {기술 스택 요약}.

## 스펙 문서
이 프로젝트는 스펙 주도 개발(Spec-Driven Development)을 따릅니다.
- 전체 스펙 목록: `.specs/INDEX.md`
- 개별 스펙: `.specs/spec_{기능명}/spec.md`
- 진행 상황: `.specs/spec_{기능명}/PROGRESS.md`

## 작업 규칙
1. 기능을 구현하기 전에 반드시 해당 스펙의 `spec.md`를 참고합니다.
2. 코드 구현이 완료되면 해당 스펙의 `PROGRESS.md`를 갱신합니다.
3. 설계 변경이 발생하면 `spec.md`를 먼저 갱신하고 코드에 반영합니다.
4. 새 파일을 생성할 때 `spec.md`의 파일 구성에 명시된 경로를 따릅니다.

## 빠른 참조
- {프로젝트 전반의 핵심 규칙}
```

### 3-4. OpenAI (ChatGPT / GPTs)

유니티 버전 시작 리소스 문서의 3-4 섹션과 동일한 구조를 따릅니다. 프로젝트 정보와 기술 스택만 실제 값으로 대체하세요.

### 3-5. Claude Projects (claude.ai)

유니티 버전 시작 리소스 문서의 3-5 섹션과 동일한 구조를 따릅니다.

## 4. 프로젝트에 추가하는 방법

### 4-1. Claude Code

1. 프로젝트 루트에 `.specs/` 폴더와 `INDEX.md`를 생성합니다.
2. `CLAUDE.md`를 3-1 템플릿으로 생성합니다.
3. (선택) `.claude/skills/`에 스킬을 추가합니다.
4. Git에 커밋합니다.

### 4-2. Codex CLI

1. 프로젝트 루트에 `.specs/` 폴더와 `INDEX.md`를 생성합니다.
2. `AGENTS.md`를 3-2 템플릿으로 생성합니다. 또는 `/init`으로 초기 파일을 생성한 뒤 커스터마이징합니다.
3. (선택) `.codex/skills/`에 스킬을 추가합니다.
4. Git에 커밋합니다.

### 4-3. Cursor

1. 프로젝트 루트에 `.specs/` 폴더와 `INDEX.md`를 생성합니다.
2. `.cursor/rules/spec-driven-dev.mdc`를 3-3 템플릿으로 생성합니다.
3. (선택) 코딩 규약 등 추가 .mdc 파일을 생성합니다.
4. Git에 커밋합니다.

### 4-4. OpenAI / Claude Projects

1. 로컬 프로젝트에 `.specs/` 폴더와 `INDEX.md`를 생성합니다.
2. 웹에서 시스템 프롬프트 또는 Custom Instructions를 설정합니다.
3. 작업 시 spec.md를 파일 첨부 또는 복사·붙여넣기로 전달합니다.

### 5-4. 참고사항

- 이미 존재하는 파일은 덮어쓰지 않고 건너뜁니다.
- 유니티 프로젝트가 아닌 경우 `Assets/` 폴더 존재 여부를 체크하지 않습니다.
- 스크립트 실행 후 생성된 파일의 플레이스홀더를 실제 값으로 대체하세요.
