# PROGRESS.md 템플릿

> 진행 상황만 담는다. 설계는 spec.md, 전체 현황은 INDEX.md. 겹쳐 쓰지 않는다.

```markdown
# 〈기능명〉 — 진행 상황

## 영역별 진행도
| 영역 | 상태 | 비고 |
|------|------|------|
| 〈데이터 모델〉 | ✅ 완료 | |
| 〈핵심 로직〉 | 🔨 진행중 | 〈무엇 작업 중〉 |
| 〈UI〉 | ⬜ 미착수 | |
| 〈서버 연동〉 | ⬜ 미착수 | 〈대기 사유〉 |

## 작업 히스토리
### 〈YYYY-MM-DD〉
- 〈한 일〉

## 진행중
- [ ] 〈현재 작업 항목〉

## 다음 할일
- [ ] 〈다음 항목〉

## 결정 기록
### 〈결정 제목〉 (〈YYYY-MM-DD〉)
- **결정:** 〈무엇으로 정했는지〉
- **대안:** 〈고려한 다른 선택지〉
- **사유:** 〈왜 이걸 골랐는지〉
```

## 작성 팁
- **상태 아이콘**: ✅ 완료 / 🔨 진행중 / ⬜ 미착수.
- **결정 기록이 핵심 자산**이다. 비자명한 기술 선택마다 결정/대안/사유 3줄을 남긴다 — 나중에 "왜 이렇게 했지?"에 답한다.
- 작업 히스토리는 날짜 역순(최신 위). AI가 갱신하면 안 한 걸 완료로 적지 않았는지 검토한다.
