# 에러 처리 / 엣지 케이스

## 필요한 컴포넌트가 캐넌 목록에 없음

`get_component`가 못 찾으면 임의로 새 클래스를 만들지 않는다. `AskUserQuestion`으로 "이 디자인시스템엔 없는 컴포넌트인데 새로 정의할까요, 있는 걸로 대체할까요?"를 묻는다.

## 카테고리 번호 · 네비게이션 구조

각 프로젝트의 상위 규칙(예: 카테고리 번호 매핑)은 사람이 관리하는 영역이다. 이 스킬은 절대 카테고리/네비게이션 구조를 수정하지 않는다 — 색·타이포·radius·spacing·컴포넌트 같은 시각/토큰 레이어만 다룬다.

## 카피 · 보이스톤

"피해야 할 단어" 같은 문구 톤 규칙은 이 스킬의 영역이 아니다. 코드에서 마주치더라도 건드리지 않는다.

## 서버 도구 계약이 일부만 일치

`server-detection.md`의 부분 지원 규칙을 따른다: 사용 가능한 도구만 쓰고, 못 쓰는 단계는 건너뛰며 어떤 기능이 제한되는지 안내한다.

## 재시도 상한(3회)에 걸린 경우

`polish-mode.md` 4단계의 절차대로 자동 수정을 멈추고 남은 위반을 파일·라인·이유와 함께 보고한다. 사용자가 계속 진행을 요청하면 재시도 카운터를 초기화하고 이어간다.
