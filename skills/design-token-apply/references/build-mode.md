# 제작 모드 (1-4단계)

요청 대상이 **새 파일이거나 빈 페이지**일 때 이 모드로 진행한다.

## 1. Discover

무엇을 만들지 파악한다: 페이지 전체인지, 컴포넌트 하나인지. 기존에 비슷한 페이지/컴포넌트가 있으면 참고용으로 읽어 레이아웃 관례를 파악하되, 토큰 값은 참고하지 않는다(2단계에서 항상 최신 스펙을 다시 질의).

## 2. Reference

만들려는 요소에 해당하는 캐넌 컴포넌트를 `get_component`로 조회한다. 이름이 명확하지 않으면 `search_guidelines`로 레이아웃/컴포넌트 가이드를 먼저 찾는다. 필요한 토큰(색·radius·spacing 등)은 `get_token`으로 개별 조회한다 — 값을 스킬 지식이나 기억에서 끌어오지 않는다.

## 3. Apply

- `scaffold` 도구로 캐넌 컴포넌트의 정확한 뼈대(HTML/JSX + 클래스 + 예시)를 받는다.
- scaffold 결과를 그대로 붙여넣지 말고, 페이지 고유 레이아웃(그리드 배치, 텍스트 내용, 이미지 등)만 사용자 코드로 채운다.
- 새 컴포넌트 클래스를 임의로 만들지 않는다. 필요한 컴포넌트가 캐넌 목록에 없으면 `references/edge-cases.md`의 절차를 따른다.
- raw hex, 임의 px, 인라인 `:root` 오버라이드를 절대 쓰지 않는다 — 전부 `get_token`으로 조회한 `var(--...)` 참조를 쓴다.

## 4. Verify

제작 모드도 완료 전 `validate_code`/lint 0 violation을 확인한다. 상세 재시도·보고 절차는 `references/polish-mode.md`의 4단계(Verify — 검증 게이트)를 그대로 따른다 — 두 모드가 같은 게이트를 공유한다.
