# 오프라인/비-git 활동 보충

git에 안 남는 활동을 **오늘** Daily에 기록한다. 회의, 외부 도구 작업, 휴식, 비-git 의사결정 등.
collect-git이 못 잡는 활동(git-less 세션·외부 미팅·휴식일)을 `<!-- manual-start -->`~`<!-- manual-end -->`
블록 안에 append 한다. 오늘 Daily가 없으면 먼저 만든 뒤 보충한다.

## 실행 절차

### 1단계: 메모 내용 확인

사용자가 알려준 메모 내용을 받는다. 불명확하면 "무엇을 기록할까요?"라고 묻고 중단.

### 2단계: 오늘 Daily 존재 확인

```bash
ls ~/Documents/Worklog/Daily/$(date +%F).md 2>/dev/null
```

- **있으면** → 3단계로.
- **없으면** → worklog 스킬 흐름에 **위임**해 오늘 Daily를 먼저 생성한 뒤 3단계로(Daily 작성 로직 중복 구현 금지).

### 3단계: 오프라인 활동 블록에 append

`## 오프라인 활동` 섹션의 `<!-- manual-start -->`와 `<!-- manual-end -->` **사이**에 메모를 한 줄(또는 소항목)로 추가한다.

```markdown
## 오프라인 활동
<!-- manual-start -->
- (기존 내용 보존)
- <새 메모 내용>          ← 새로 추가
<!-- manual-end -->
```

규칙:
- **기존 manual 블록 내용은 보존**하고 끝에 덧붙인다(덮어쓰기 금지).
- placeholder 안내 문구(`(공란 — 회의·외부 도구 작업·휴식 등은 wl-note로 추가)`)만 있으면 그 줄을 새 메모로 **교체**한다.
- 섹션·마커가 없으면(구형 일지) `## 프로젝트 연결` 앞에 `## 오프라인 활동` + 마커 블록을 만들고 추가한다.

### 4단계: 저장 확인

추가한 줄과 파일 경로를 사용자에게 알린다. 필요 시 `## 프로젝트 연결`에 관련 `[[Projects/노드]]`를 보충하도록 제안한다(레포명 직박 금지, 실재 노드만).

## 검증 체크리스트

- [ ] 오늘 날짜(`date +%F`) Daily에 기록했는가? (과거 날짜는 wl-backfill 소관)
- [ ] `<!-- manual-start -->`~`<!-- manual-end -->` **사이**에 들어갔는가?
- [ ] 기존 오프라인 활동 내용을 보존했는가? (덮어쓰지 않음)
- [ ] 오늘 Daily가 없던 경우 worklog 스킬로 먼저 생성했는가?
