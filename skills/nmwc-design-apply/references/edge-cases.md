# 에러 처리 / 엣지 케이스

## 필요한 컴포넌트가 캐넌 목록에 없음

`get_component`가 못 찾으면 임의로 새 클래스를 만들지 않는다. `AskUserQuestion`으로 "이 디자인시스템엔 없는 컴포넌트인데 새로 정의할까요, 있는 걸로 대체할까요?"를 묻는다.

## 카테고리 번호 · 네비게이션 구조

## 카피 · 보이스톤

`SKILL.md`의 "스코프 경계" 섹션 참조 — 이 스킬의 범위 밖이다.

## 서버 도구 계약이 일부만 일치

`server-detection.md`의 부분 지원 규칙을 따른다: 빠진 도구가 `scaffold`(제작 모드 Apply)나 `suggest_token`(폴리싱 모드 Apply)이면 해당 모드 전체를 사용 불가로 선언한다. 그 외 도구(`search_guidelines` 등)만 빠졌다면 그 기능만 건너뛰고 어떤 기능이 제한되는지 안내한다.

## 재시도 상한(3회)에 걸린 경우

`polish-mode.md` 4단계의 절차대로 자동 수정을 멈추고 남은 위반을 파일·라인·이유와 함께 보고한다. 사용자가 계속 진행을 요청하면 재시도 카운터를 초기화하고 이어간다.
